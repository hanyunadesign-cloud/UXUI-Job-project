# PRD: {UXUI Job}

## 1. 한 줄 정의
UXUI 디자이너 구직자가 분산된 채용공고로 인해 공고를 놓치거나 찾기 힘든 일을 해결하도록 공고모음과 알림을 제공하는 웹앱

## 2. 문제와 타깃
- 문제: 1. UXUI 채용공고가 여러 사이트에 분산되어 공고 업로드를 모니터링하는게 어렵다. 2. 내가 원하는 직무, 도메인, 스테이지에 맞는 채용 공고를 서칭하는 과정이 오래 걸린다.
- 타깃: 이직 및 취업을 준비하며 UXUI 채용 공고를 서칭 중인 UXUI 디자이너 구직자

## 3. 핵심 시나리오
- 진입: url 접속, 간단한 온보딩
- 로그인: 구글계정으로 로그인
- 채용공고 탐색: 홈에서 채용공고들을 탐색
- 공고 선택: 마음에 드는 공고 선택 후 상세페이지로 이동
- 결과 확인: 상세페이지에서 공고 내용과 나란히 노출된 AI 분석 키워드(필요 역량 키워드, 포폴과 이력서에서 어필해야 할 부분) 확인
- 다음행동: 관심있게 봤던 채용공고 마이페이지에 저장하기

## 4. 기능 범위
- Must: 구글계정으로 로그인 / 공고 상세페이지 진입 시 AI 분석 키워드 자동 노출 / 관심 채용공고 저장하기 버튼
- Won't: 지원 현황 관리(지원함/서류통과/면접 등 트래킹) / 구글 외 소셜 로그인(카카오 등) / 다국어 지원

## 5. 화면 구성
- 화면 1 · 첫화면: 시작하기 버튼, 서비스 로고, 서비스 문구
- 화면 2 · 로그인: 구글계정으로 로그인
- 화면 3 · 온보딩:
  - 관심 직무(GUI 디자이너, UXUI 디자이너, 프로덕트 디자이너, UX 라이터, UX 리서처 중 선택)
  - 관심 플랫폼(웹, 앱, 태블릿, 워치/웨어러블, 모빌리티, 가전, VR/AR 중 선택)
  - 관심 산업(커머스, 핀테크, SNS, 여행/로컬, B2B SaaS, 헬스케어, 모빌리티 중 선택)
  - 목표 스테이지(스타트업, 유니콘, 중견 기업, 대기업, 에이전시 중 선택)
  - 4개 설정. 각 설정 단계별로 skip 버튼, 이메일로 공고 알림 받기 버튼
- 화면 4 · 홈: 채용공고들, 필터
- 화면 5 · 공고 상세페이지: 공고 내용, 기업로고 사진, AI 분석 키워드 영역(핵심 역량 키워드 + 어필 포인트, 자동 노출), 지원 페이지로 이동 버튼(클릭 시 채용공고 원본 사이트로 이동), 저장하기 버튼
- 화면 6 · 마이페이지: 저장한 채용공고, 알림

## 6. AI 동작 정의
- 트리거: 사용자가 공고 상세페이지에 진입했을 때 자동 실행 (분석 결과가 이미 캐시되어 있으면 재실행 없이 캐시된 결과 노출)
- 입력: 페이지 진입 시점에, 그 공고에 등록된 원본 텍스트(그대로 복붙, 최대 3,000자) 또는 원본 이미지 1장을 그대로 API에 전달
- 출력: JSON { "core_keywords": [...], "resume_tip": "두 문장 이내 어필 포인트" }
- 프롬프트 방향: 커리어 코치 역할, 간결하고 실용적인 톤, 공고에 없는 내용 추측 금지
- 실패 처리: 분석 실패 시 키워드 영역에 "지금은 분석이 어려워요" 안내 + 재시도 버튼 노출. 원본 공고는 분석 성공 여부와 무관하게 항상 그대로 노출

## 7. 기술 스택
Next.js + Vercel 배포. 그밖의 필요한 기술 스택은 에이전트가 추천.

## 8. 디자인 톤
미니멀, 깔끔, 뉴트럴 톤(화이트/그레이/블랙) 조합, 그리드 기반의 정돈된 레이아웃 (다크모드는 만들지 말것)

- [ ] M2: API 연결 (JD 분석 AI API 연결 + 이메일 발송 연동)
- [ ] M3: Vercel 배포
